## Layer 4: Network & Connectivity

**CISA Pillars:** Networks, Devices (contributing), Data (contributing)

### Threats Addressed

Network and connectivity threats exploit the residual trust placed in corporate network adjacency — the assumption that a device on the corporate LAN is a safe device. Key threats include Man-in-the-Middle via LLMNR/NBT-NS poisoning and SMB relay (enabling pass-the-hash without valid credentials), DNS hijacking and DNS tunnelling for C2 command and control that traverses firewalls undetected, rogue Wi-Fi and evil twin access points targeting remote and travelling employees of the bank, TLS downgrade and certificate spoofing attacks against financial platform connections (Bloomberg Terminal, SWIFT, counterparty portals), IPv6 Router Advertisement injection enabling traffic redirection without touching IPv4 network controls, lateral east-west movement across a flat corporate LAN following initial compromise of a single endpoint, and data exfiltration via permitted protocols (DNS, HTTPS, QUIC) that bypass conventional perimeter inspection. Financial sector precedents: SWIFT messaging platform MitM and unauthorised fund transfer initiation, BGP-based traffic hijacking targeting financial routing infrastructure, and attacker persistence via DNS-based C2 in APT campaigns targeting tier-1 banks.

### ZTA Integration

Layer 4 network controls operate across all three NIST SP 800-207 planes and form a critical intermediate enforcement layer between the device trust signals produced by Layer 1–3 controls and the data and application access decisions protected by Layers 5–15.

**Trust Signal Generation (PDP inputs — PIP layer):** NC-03 (network location classification) produces a trust context signal — distinguishing corporate-managed, ZTNA-brokered, residential, and anonymous network origins — that the Conditional Access PE evaluates alongside device compliance and user identity. The network location signal modulates access scope: an authenticated user on a known corporate network with a compliant device receives full access; the same user on an anonymous proxy network triggers an Adaptive step-up requirement. NC-04 (encrypted DNS compliance) and NC-14 (host firewall compliance state) are surfaced as MDM compliance attributes feeding the PE.

**Policy Enforcement (PEPs):** NC-01 (ZTNA) is the primary network-layer PEP — every application access request is brokered through the ZTNA service, which enforces a combined identity + device compliance + network context decision for each individual application session, replacing the VPN model where authentication grants blanket LAN access. NC-02 (micro-segmentation) enforces a Hard gate at the network control plane between endpoint segments, preventing east-west lateral movement regardless of whether the initiating device has a valid active session with the ZTNA broker. NC-05 (DNS RPZ filtering) and NC-09 (SSL inspection) act as inline SWG PEPs intercepting malicious communication before connections are established or data is transmitted.

**Policy Administration (PAP):** NC-07 (TLS version mandate), NC-08 (CA trust store governance), NC-10 (LLMNR/NBT-NS disable), NC-11 (SMBv1/NTLM disable), and NC-12 (IPv6 RA hardening) constitute the network-layer Policy Administration controls — reducing the attack surface that PEPs must defend, and eliminating protocol-level vulnerabilities that would otherwise allow attackers to bypass PEP enforcement entirely. NC-15 (egress allowlisting) and NC-13 (QUIC blocking) close inspection bypass channels that would allow data to leave the organisation beneath the SSL inspection PEP.

**Continuous Monitoring (Management Plane):** NC-06 (DNS anomaly detection), NC-18 (NDR + endpoint network telemetry), and NC-16/NC-17 (SWG + CASB telemetry) fulfil the NIST SP 800-207 §3.5 continuous monitoring requirement at the network layer — forwarding telemetry to the SIEM and triggering SOAR-automated device isolation, Conditional Access revocation, or network quarantine when C2, lateral movement, or data exfiltration patterns are detected. Critically, NDR sensor visibility persists even if the endpoint EDR agent is disabled — this complementary coverage is the architectural reason both telemetry sources are required rather than treating them as redundant.
