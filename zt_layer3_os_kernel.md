## Layer 3: Operating System & Kernel

**CISA Pillars:** Devices, Applications & Workloads (contributing)

### Threats Addressed

Operating system and kernel threats target the layer that sits directly above firmware and provides the runtime environment for every security control above it — making kernel compromise the most impactful single attack class after firmware. Key threats include kernel exploitation via unpatched OS vulnerabilities enabling privilege escalation to SYSTEM/root (CVE-class zero-days, BYOVD — Bring Your Own Vulnerable Driver), rootkit persistence that survives OS reinstallation by operating below the operating system's own kernel monitoring capabilities, Direct Memory Access attacks via kernel-mode drivers, LSA (Local Security Authority) credential harvesting from LSASS memory, Credential Guard and VBS bypass via misconfiguration, application allow-listing bypass through MSIX/AppX packaging and script-based Living Off the Land (LOLBAS) techniques, and unpatched local privilege escalation vulnerabilities exploited post-initial-access to elevate from standard user to SYSTEM. Financial sector relevance: kernel-level compromise on a SWIFT operator workstation provides a threat actor with the memory-resident access needed to tamper with transaction data in transit between the endpoint and the SWIFT messaging gateway, the highest-impact single control plane failure in financial services endpoint security.

### ZTA Integration

Layer 3 OS and kernel controls form the foundational trust platform on which every higher-layer security control depends. The integrity of the OS layer is a prerequisite for trusting any signal produced by it — an OS-level attacker can falsify EDR telemetry, manipulate MDM compliance reports, and forge the device posture signals that the Conditional Access PE evaluates.

**Trust Signal Generation (PDP inputs):** OS-01 and OS-03 (OS version) provide the device compliance signal that produces a Hard gate CA response for unsupported OS versions. OS-02 and OS-04 (CIS baseline compliance) generate Soft gate compliance signals — deviation from the hardening baseline is surfaced to the PE and restricts access scope rather than fully blocking. OS-16 and OS-17 (patch compliance) produce patch currency signals that feed the PE with Soft gate posture for overdue endpoints.

**Local Policy Enforcement (PEPs):** VBS and HVCI (OS-05) enforce kernel code integrity as a hypervisor-resident PEP that cannot be tampered with from kernel mode. Credential Guard (OS-06) enforces LSA credential isolation independently of SIEM or cloud connectivity. WDAC (OS-09) enforces application execution control as a kernel-resident PEP. ASR rules (OS-08) enforce specific execution behaviour restrictions at the kernel driver level. SIP (OS-11) and Gatekeeper (OS-12) enforce macOS execution trust as OS-resident PEPs.

**Policy Administration (PAP):** LAPS (OS-14) and local admin governance (OS-15) are PAP controls that configure the privilege model the PEPs operate within. Patch management (OS-16/17/18) is the most critical PAP function at this layer — ensuring that the kernel the security controls trust is not itself the attack vector via a known, exploitable CVE.

**Continuous Monitoring (Management Plane):** XProtect/ESF telemetry (OS-13), WDAC block events (OS-09), ASR block events (OS-08), and CIS compliance deviation reports all contribute to the SIEM management plane. Kernel-level events (driver loads, process launches, anomalous system calls) provide the highest-fidelity endpoint signals available for detection of sophisticated attacker techniques including BYOVD, process injection, and LOLBAS abuse.
